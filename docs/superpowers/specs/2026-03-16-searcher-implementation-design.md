# Searcher Tool Implementation Design (Firecrawl Based)

## 1. Overview
The Searcher is a standalone tool designed to fetch AI Engineer job postings from recruitment platforms (initially Nowcoder/牛客网) using Firecrawl's scraping and LLM parsing capabilities.

## 2. Interface Definition
- **Tool Name**: `unified_search_tool`
- **Arguments**:
  - `query` (string): The search keyword generated by the Planner.
  - `platform` (string): Target recruitment site (default: "nowcoder").
- **Output**:
  - `List[Dict]`: A list of objects containing `title`, `url`, and a brief `description`.

## 3. Detailed Logic Flow
1. **URL Construction**: Construct the platform-specific search URL (e.g., `https://www.nowcoder.com/search-job?query=...`).
2. **Firecrawl Scraping**: Use `FirecrawlApp.scrape_url` to get the page content in Markdown format.
3. **LLM Extraction (Schema)**: 
   - Pass the Markdown content to a structured extractor (OpenAI Function Calling or Pydantic Output Parser).
   - Use a prompt specifically tuned to extract job list entries even from complex DOM-to-Markdown noise.
4. **Validation**: Ensure all extracted items have valid URLs.

## 5. Security & Configuration
- **API Keys**: Firecrawl API key must be provided via `.env`.
- **Rate Limiting**: Firecrawl handles headful rendering and retry logic, reducing local implementation complexity.
