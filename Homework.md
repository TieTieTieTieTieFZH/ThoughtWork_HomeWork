# 校招 AI Engineer 面试题

## 🎯 场景设定

你是一名正在找工作的应届毕业生，目标岗位是 **AI Engineer（校招/实习）** 。
每天手动浏览招聘网站效率太低，你决定构建一个 **Agentic AI 求职助手** ，自动帮你搜索、筛
选、整理岗位信息。

## 📌 核心任务

```
请设计并实现一个 基于 Agent 的自动求职系统 ，
自动收集 50 条 AI Engineer 校园招聘岗位信息 。
```
岗位要求：

## 🤖 技术实现必须包含：Agentic AI

你不能只写一个普通爬虫脚本，而是需要构建一个 **具备“自主规划与执行能力”的 AI Agent 系
统** 。

## Agent 需要具备以下能力

#### 能力 说明

#### 🧭 任务规划 将“找到 50 个岗位”拆解成搜索 → 抓取 → 解析 → 去重 → 存储

#### 🌐 工具调用 能调用搜索工具、网⻚抓取工具、信息解析工具

#### 🔁 迭代搜索 当岗位数量不足时，自动调整关键词继续搜索

```
🧠 语义判断 判断岗位是否属于 AI Engineer 方向
🧹 数据清洗 自动结构化岗位信息
📦 结果汇总 输出标准化 JSON/CSV
```
## 🏗 推荐 Agent 架构（仅供参考）

#### 面向应届生 / 校招 / 实习

#### 与 AI / 机器学习 / 大模型 / 数据智能 / 算法工程 相关

#### 来自 至少 2 个不同招聘网站


#### 可使用（但不限于）：

### 示例 Agent 工作流程

```
User Goal: 找到 50 个AI Engineer校招岗位 ↓ Agent 规划 1. 选择招聘网站 2. 构造搜索关键
词（AI Engineer / 机器学习工程师 / 算法工程师 / LLM Engineer 等） 3. 抓取职位列表⻚
```
4. 进入详情⻚提取信息 5. 判断是否符合“AI Engineer + 校招/实习” 6. 去重 7. 不足 50 条 →
调整关键词继续搜索 8. 输出结构化结果

## 🗂 输出数据结构

#### 字段 说明

```
title 职位名称
company 公司名称
location 工作地点
salary 薪资范围
tech_tags 技术关键词（如 LLM / CV / NLP / 推荐系统）
requirements 岗位核心技能摘要
source 招聘网站
job_url 岗位链接
```

## 🎤 面试官可能追问

## 🧩 这道题真正想考什么？

#### 这题不是在考“会不会写爬虫”，而是在考：

✅ 会不会用 **LLM + Tool 构建可执行 Agent**
✅ 会不会让 AI **自主拆解并完成真实复杂任务**
✅ 是否具备 **AI 工程化 + Agent 设计能力**

1. Agent 如何判断岗位是真正的 AI Engineer，而不是普通后端工程师？
2. 如果搜索结果不足 50 条，Agent 如何自动扩展搜索范围？
3. 你如何防止 Agent 在搜索时进入无限循环？
4. 工具调用失败时 Agent 如何 fallback？
5. 如果要扩展到“数据科学岗位”，需要改哪些模块？
